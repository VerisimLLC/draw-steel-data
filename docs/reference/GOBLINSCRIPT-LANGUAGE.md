# GoblinScript Language Reference (verified against the C# parser)

Authoritative reference generated by reading the parser/compiler source:
`D:/dev/dmhub/Assets/Scripts/GoblinScript.cs` (3370 lines).
Cross-checked against the Lua wrapper `DMHub Utils/GoblinScript.lua`

## 1. Big Picture

GoblinScript is a side-effect-free expression language. Every formula evaluates
to a number, boolean, string, or opaque object. Two execution paths exist
(`GoblinScript.cs:489` `Eval(...)`):

1. **Reduce path** -- the parser's `Expression.Reduce(ctx)` walks the AST and
   tries to compute a `LuaValue` directly, using `ResolveSymbolsToObject` and
   the symbol-lookup function. Used when no compiled form is available.
2. **Compile path** -- `CompileToLua` emits Lua source (visible in the console
   when `g_useCompiled` is true, default on). The Lua function is cached
   (`s_GoblinScriptCache`, capped at 4000 entries before full eviction
   `GoblinScript.cs:677`) and re-run for each invocation. Lua-side cache lives
   in `GoblinScript.lua:19` `g_compiled`.

The Lua-side wrapper `ExecuteGoblinScript(formula, symbols, defaultValue,
contextMessage)` (`GoblinScript.lua:134`) is the entry point used everywhere
in the codex. `formula` may be a number (returned directly) or a string. If a
string, it is compiled once via `dmhub.CompileGoblinScriptDeterministic` and
cached. If compilation fails, the formula is permanently set to `false` in the
cache and `defaultValue` is returned forever (the cache is only flushed via
`/flushcompiledgoblinscript`).

`ExecuteGoblinScript` post-processes booleans: `true -> 1`, `false -> 0`. So
**every formula returns a number** in practice, even Boolean-shaped formulas.

## 2. Tokens (`GoblinScript.cs:953-1176`)

The lexer is regex-based. Token types in priority order, with their regexes:

| Token | Pattern | Notes |
|---|---|---|
| `ADD` | `+` | binary or unary |
| `SUB` | `-` | binary or unary |
| `MULT` | `*` | |
| `DIV` | `/` | always integer-divides (`math.floor`) |
| `IS_NOT` | `is not` (case-insensitive) | requires surrounding space |
| `IS` | `is` | requires surrounding space |
| `HAS_NOT` | `has not` | requires surrounding space |
| `HAS` | `has` | requires surrounding space |
| `NOT` | `not` | unary, requires trailing space |
| `KEYWORD` | `advantage|disadvantage|novantage|keep|drop|exploding|reroll|keep high|keep low|drop high|drop low|kh|kl|dh|dl` | dice modifiers; **elided in deterministic mode** so they will not parse in non-roll contexts |
| `LPARENS` / `RPARENS` | `(` `)` | |
| `LARROW` | `<-` | |
| `COMMA` | `,` | |
| `CATEGORY` | `[anything]` | metadata tag, captured as `Token` |
| `D` | `d` followed by digit/space | dice die marker |
| `WHEN` | `when` | conditional |
| `WHERE` | `where` | substitution binding |
| `ELSE` | `else` | conditional fallback |
| `AND` | `and` | logical |
| `OR` | `or` | logical / numeric max |
| `CMP` | `>=`, `<=`, `>`, `<`, `=`, `<>`, `!=`, `~=` | numeric or string comparison |
| `NUM` | `[0-9]+\.?[0-9]*` | |
| `STRING_LITERAL` | `"..."` | no escapes -- you cannot put a `"` inside |
| `DOT` | `.` | object field access |
| `IDENTIFIER` | `[a-z]+` (case-insensitive) | merges adjacent IDENTIFIER tokens, see below |

**Comments** are `{ ... }` (curly braces, balanced). The entire content
including braces is dropped before parsing (`GoblinScript.cs:1034-1059`). They
work anywhere whitespace works.

**Identifier merging** (`GoblinScript.cs:1006-1014`): consecutive IDENTIFIER
tokens with whitespace between them are concatenated into one identifier.
That is why `Maximum Stamina`, `Power Roll Bonus`, `Movement Speed` are all
single symbols. The whitespace becomes part of the merged token but is then
stripped at lookup time (lookups normalize via `ToLower().Replace(" ", "")` at
`GoblinScript.cs:1204`, `820`, `846`, `871`).

**Word matching gotcha:** `is`, `has`, `not`, `and`, `or`, `when`, `else`,
`where` are keyword regexes that come BEFORE the identifier regex. If you
write a symbol named `Andromeda`, the lexer will eat the `and` first.
Workaround: rename, or use string/dot context.

**Deterministic mode** (`deterministic = true`, set by
`dmhub.CompileGoblinScriptDeterministic`) skips dice keywords entirely. All
codex compile sites we found use deterministic mode.

## 3. Operator Precedence

From `Expression.ParseExpression` (`GoblinScript.cs:1215-1392`), lowest
precedence first (i.e. parsed first, binds last):

1. `where` (`TryParseWhere`, evaluated by lifting the binding before any
   evaluation -- see Section 5)
2. `+`, `-` (left-associative; unary check via `s_unaryCheckPrevTokenTypes`)
3. `when` / `else` (ternary; `else` can appear at most once after each `when`)
4. `or`, `and` (handled together but `OR` is checked first, so `and` binds
   tighter than `or` -- standard precedence)
5. `=`, `~=`, `!=`, `<>`, `>`, `<`, `>=`, `<=` (`CMP`)
6. `*`, `/`
7. Unary `+`/`-` (only at start of an operand)
8. `is`, `is not`, `has`, `has not`
9. Function call (must end in `)` and have a matching `(`)
10. `keep`, `drop`, dice keywords (`KEYWORD`, `D`)
11. `.` (right-associative -- `a.b.c` parses as `a.(b.c)`, see
    `TryParseOperator` `associativity=true`)

Outermost parens are stripped first (`GoblinScript.cs:1270-1293`). `[category]`
is parsed even before `where` (`GoblinScript.cs:1296-1316`) and divides the
expression on either side of the bracket.

**Surprise from prose:** the prose guide says `*`, `/` is precedence 6 and
comparisons 5, but the parser parses comparisons FIRST (lower precedence),
then `*`, `/`. Both descriptions agree on binding tightness; only ordering
language differs.

## 4. Type Model

GoblinScript values inherit Lua's types but are normalized on use. The
compiler tracks a `luaType` for each stack entry: one of `"number"`,
`"boolean"`, `"string"`, `"function"`, or `nil` (unknown/dynamic). Helpers
`EnsureNumber`, `EnsureBoolean`, `EnsureString`, `EnsureStringIsLowercase`
inject coercion at the use site (`GoblinScript.cs:183-269`).

| Operation | Coerces operands to | Result type |
|---|---|---|
| `+`, `-`, `*` | number (`tonumber(x) or 0`) | number |
| `/` | number, then `math.floor` | number (always integer) |
| `>`, `<`, `>=`, `<=` | number | boolean |
| `=`, `~=` | dynamic dispatch (number/string/object) | boolean |
| `and`, `or` | boolean (numeric `<= 0` => false) | boolean, EXCEPT `or` of two numbers => `math.max(a, b)` |
| `not` | boolean (number `<= 0` => false) | boolean |
| `is`, `is not` | string (lowercased) | boolean |
| `has`, `has not` | string (lowercased substring) | boolean |
| `when` ... `else` | RHS coerced to boolean for the test | type of selected branch (or `0` if no `else`) |

**Falsy** (`GoblinScript.cs:213-222`, `2227-2228`): `false`, `nil`, any number
`<= 0`. So `Stamina when Hidden This Turn else 0` works because boolean
`true` and number `> 0` are both truthy.

**`or` returning max**: this is real and used heavily. `Stamina or Level or
4` returns `max(stamina, level, 4)` if all are numbers (verified by unit
tests in `GoblinScript.lua:344-345`). If either operand is a non-number, it
falls back to short-circuit boolean `or` (returning `1`/`0`).

**String compare normalization** (`GoblinScript.cs:2532-2534`): both sides
have `string.lower()` and `string.gsub(" ", "")` applied. So
`"Power Roll" = "powerroll"` is true. This applies to `=`, `~=`, `is`,
`is not`, `has`, `has not`.

**Object equality** (`GoblinScript.cs:2629-2654`): `=` between two table or
function values resolves both via `ResolveSymbolsToObject`, then compares
tables by reference (`Table.Equals`). Unit test
`"symbols = self" -> 1` verifies this works for self-comparison.

**Set membership** (`has`): when the LHS reduces to a `function` (i.e. a
symbol-lookup function), the parser tries to call it with the magic key
`"__is__"` to get a custom membership predicate
(`GoblinScript.cs:2412-2424`). `Conditions has "Prone"` works because the
StringSet returned by the `conditions` lookup has an `__is__` field that
implements substring/exact lookup. When `is` rather than `has` is used on a
function/object, the same `__is__` machinery dispatches.

## 5. `where` Bindings

`where x = expr` registers a textual substitution: every later occurrence of
`x` in the formula is replaced (post-parse) by the parsed `expr`. Implemented
by `Expression.ParseExpressionResolveWhere` -> `VisitWhereExpressions` ->
`SubstituteWhere` (`GoblinScript.cs:1186-1212`, `2880-2945`).

```
Stamina + x where x = 7
  -> Stamina + 7  (verified, GoblinScript.lua:346)
```

Limitations:
- `where` is parsed first (lowest precedence), so the binding scope is the
  whole expression.
- Only one binding per `where`; chain via `where x = ... where y = ...`
- The substituted text is the parsed sub-expression, not the textual source,
  so re-substitution does NOT happen recursively.

## 6. Conditional `when` / `else`

`value when condition` returns `value` if `condition` is truthy, else `0`
(`GoblinScript.cs:2236-2243`). With `else`, the else-branch is returned
instead. Type inference is conservative: `entry.luaType` is only set if both
branches agree (`GoblinScript.cs:2200-2208`).

```
2 + 5 when stamina > 5 and level = 1 else 12
  -> 7 when stamina>5 & level=1, else 14  (GoblinScript.lua:341-342)
```

Multiple `when`s: an unmatched `when` consumes a single `else`; the
matching is greedy-left (`TryParseWhen` `GoblinScript.cs:1528-1584`). For
chained "if/elif/else", parenthesize.

## 7. Functions

Function call syntax: `name(arg1, arg2, ...)`. The parser identifies a call
by matching a trailing `)` to a `(` and treating the LHS as the function
expression (`TryParseFunctionCall` `GoblinScript.cs:1395-1468`).

**Built-in functions** (`GoblinScript.cs:3306-3314` -- the C# table; backed
in Lua by `BuiltinGoblinScriptFunctions` at `GoblinScript.lua:225-258`):

| Function | Signature | Notes |
|---|---|---|
| `min` | `min(a, b, ...)` | variadic, returns smallest castable-to-number arg |
| `max` | `max(a, b, ...)` | variadic, returns largest |
| `floor` | `floor(x)` | rounds down |
| `ceiling` | `ceiling(x)` | rounds up |
| `Friends` | `Friends(creature_a, creature_b)` | true if both tokens exist and `IsFriend`. Args may be creature symbols or token IDs. Unwraps function symbols via `ResolveSymbolsToObject`. |
| `Line of Sight` | `Line of Sight(a, b, [pierceSurfaces])` | `1` clear, `0.5` half cover, `0.25` three-quarters, `0` full/blocked. `pierceSurfaces` is an integer count of walls the cover calc may ignore. |
| `Substring` | `Substring(haystack, needle)` | case-insensitive substring containment, returns `1`/`0` |

**Symbol-provided functions** (the dot-access pattern): a creature symbol
that returns a function value can be called. Examples that come up
constantly:

```
Self.Distance(Target)
Caster.Count Nearby Enemies(5, "Goblin")
Conditions has "Prone"            { uses __is__ on the StringSet }
Self.Stacks("Bleeding")
Cast.HasTarget(Self)
```

These are not "built-in" -- they are creatures'/casts' symbol entries that
return a Lua closure. See `GOBLINSCRIPT-SYMBOLS-AUTO.md` for the catalog.

**Note on case:** function names are lowercased before lookup
(`GoblinScript.cs:1457`). Identifiers used as bare symbols are also
lowercased and space-stripped. So `Friends`, `friends`, `FRIENDS`,
`f r i e n d s` (after merge of identifiers) all hit the same key.

## 8. Dot Access

`Self.Stamina`, `Cast.Spaces Moved`, `Resources.Heroic Resource` all use the
dot operator. Right-associative: `a.b.c` parses as `a.(b.c)` and evaluates
left-to-right by swapping `symbols` (`GoblinScript.cs:2082-2118`). The
compiler emits roughly:

```
backup = symbols
symbols = sym_a       { evaluate LHS }
if type(symbols) == 'table' then
  symbols = GenerateSymbols(symbols)
end
sym_b = symbols("b")  { evaluate RHS in new scope }
symbols = backup
```

If the LHS evaluates to a `table`, `GenerateSymbols(table)` is called to
materialize it into a lookup function. So `Resources` returns a
`CharacterResourceCollection` table; `Resources.Heroic Resource` then
dispatches the key `"heroicresource"` to that collection's `lookupSymbols`
(`Resource.lua:1095-1122`).

**Resource lookup case insensitivity is total**: the resource symbol name is
generated via `string.gsub(string.lower(resourceInfo.name), "%W", "")`, which
strips all non-alphanumerics. So `Resources.Heroic Resource`,
`Resources.heroicresource`, and `Resources.HeroicResource` all match -- but
**bare `Heroic Resource` does NOT work** (you must go through `Resources.`).
This trips people up; see audit doc.

## 9. Dice Notation (non-deterministic only)

Dice tokens (`d`, `keep`, `drop`, etc.) are stripped from the token stream in
deterministic mode. They only parse when `deterministic=false`, which in
practice is just for roll formulas evaluated by the dice/roll system.

Syntax mirrors standard RPG roll notation: `2d10`, `1d20 keep high`,
`4d6 drop low`. The category bracket `[fire]` attaches a tag to a roll
expression. Documented in `RollParser.KeywordDocs`; not duplicated here.

## 10. Special Behaviors and Gotchas

### Symbol resolution is dynamic / lazy

`IdentifierExpression.Compile` (`GoblinScript.cs:1848-1867`) emits
`sym_x = symbols("x")` at compile time. The actual lookup happens on every
invocation because the same compiled function is cached and reused with
different `symbols` callbacks.

### `Self` is special only if registered

The `self` symbol is registered in every type's `lookupSymbols` table
(`Creature.lua:7566-7568`, `ActivatedAbility.lua:5067-5069`,
`PathMoved.lua:6135-6137`, etc.). Bare `Self` returns the subject. There is
no parser-level `Self` keyword; it is just a convention.

### Recursion is capped at depth 10

`s_recursionDepth > 10` (`GoblinScript.cs:512-527`) bails out and returns
`"0"`. Triggered when one formula's evaluation calls another (e.g.
`activationCondition` for a power-roll modifier reading `Cast.Damage Dealt`
which itself triggered a modifier).

### Per-frame resource recursion guard

`Resource.lua:1106-1113` `g_resourceLookupRecursionProtection` prevents
`Resources.X` from recursing into itself within the same engine frame.

### `Cast` is mutated mid-evaluation

`ActivatedAbilityCast` is a single mutable object handed around through
`options.symbols.cast`. Reading `Cast.Damage Dealt` from a per-tier
formula returns the running total at that point. Order of evaluation
matters; consult call sites if you depend on a specific value.

### Comparisons against bare identifiers

`Type is Goblin` works without quoting `Goblin` (`GoblinScript.cs:2351-2360`):
when the RHS is an `IdentifierExpression` whose value resolves to nil, the
compiler substitutes the literal symbol text as a string. So `is` / `is not`
/ `has` / `has not` accept either `"Goblin"` or `Goblin`. This does **not**
work for `=` / `~=`.

### `not` returns boolean

`not Stamina` evaluates `Stamina <= 0` (because numeric falsy). Useful for
`not Hidden This Turn`.

### Compilation errors are silent at runtime

A formula that fails to compile is cached as `false` and silently returns
`defaultValue` from then on. The error is logged via `print(...)` (visible
in console only). To force recompile after fixing, run
`/flushcompiledgoblinscript`.

### Empty / nil formulas

`ExecuteGoblinScript("", ...)` and `ExecuteGoblinScript(nil, ...)`
short-circuit and return `defaultValue` immediately
(`GoblinScript.lua:135-137`).

### Numeric formulas pass through

If you pass a number where a formula is expected, it is returned directly
without compilation (`GoblinScript.lua:139-141`). YAML int values for `range:
5` therefore work without quoting.

### The `where` cascade is parsed at most once

Multiple `where` clauses build up a substitution dict; a binding cannot
reference an earlier binding (substitution is post-parse, so the parsed
expression for `x = a + 1 where a = 7` does not re-substitute).

### `debug` suffix toggles tracing

`Eval(...)` (`GoblinScript.cs:529-534`) checks if the input ends with
`debug` and, if so, strips it and sets `formulaDebugging = true`. Useful for
ad-hoc tracing in the console.

## 11. Compile Output Inspection

The compiler emits Lua source visible via `Debug.Log`
(`GoblinScript.cs:469`). To see what your formula compiles to, look in the
DMHub console after triggering the formula (e.g. via the GoblinScript
debugger panel or by editing a field).

`Self.Stamina + Maximum Stamina/2` compiles roughly to:

```lua
return function(symbols)
  local sym_self,backup,result,sym_stamina,sym_maximumstamina,value
  sym_self = symbols("self")
  backup = symbols
  symbols = sym_self
  if type(symbols) == 'table' then
    symbols = GenerateSymbols(symbols)
  end
  sym_stamina = symbols("stamina")
  symbols = backup
  result = sym_stamina
  sym_maximumstamina = symbols("maximumstamina")
  value = sym_maximumstamina
  value = tonumber(value) or 0
  value = math.floor(value / 2)
  result = result + value
  return result
end
```
